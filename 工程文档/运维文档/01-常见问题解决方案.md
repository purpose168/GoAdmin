## 1. 安装与配置问题

### 1.1 Go 环境问题

#### 问题 1.1.1: Go 版本不兼容

**问题描述**:
```
go: cannot find main module, but .go.mod file was found
```

**解决方案**:

1. 检查 Go 版本:
```bash
go version
```

2. 确保使用 Go 1.24.2 或更高版本:
```bash
# 升级 Go
wget https://go.dev/dl/go1.24.2.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.24.2.linux-amd64.tar.gz
```

3. 清理模块缓存:
```bash
go clean -modcache
go mod download
```

#### 问题 1.1.2: GOPATH 配置错误

**问题描述**:
```
cannot find package "github.com/purpose168/GoAdmin" in any of:
    /usr/local/go/src/github.com/purpose168/GoAdmin
```

**解决方案**:

1. 设置 GOPATH:
```bash
echo 'export GOPATH=$HOME/go' >> ~/.bashrc
echo 'export PATH=$PATH:$GOPATH/bin' >> ~/.bashrc
source ~/.bashrc
```

2. 验证 GOPATH:
```bash
go env GOPATH
```

#### 问题 1.1.3: Go 代理配置问题

**问题描述**:
```
go: github.com/purpose168/GoAdmin@v1.0.0: reading https://sum.golang.org/lookup/...: dial tcp: lookup sum.golang.org on 10.0.2.3:53: server misbehaving
```

**解决方案**:

1. 配置 Go 代理:
```bash
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy,https://goproxy.cn,direct
```

2. 或者使用七牛云代理:
```bash
go env -w GOPROXY=https://goproxy.cn,direct
```

3. 验证代理配置:
```bash
go env GOPROXY
```

### 1.2 依赖问题

#### 问题 1.2.1: 依赖下载失败

**问题描述**:
```
go get github.com/purpose168/GoAdmin: module github.com/purpose168/GoAdmin: Get "https://proxy.golang.org/github.com/purpose168/GoAdmin/@v/list": dial tcp: lookup proxy.golang.org: no such host
```

**解决方案**:

1. 配置 Go 代理:
```bash
go env -w GOPROXY=https://mirrors.aliyun.com/goproxy,https://goproxy.cn,direct
```

2. 清理模块缓存:
```bash
go clean -modcache
```

3. 重新下载依赖:
```bash
go mod download
```

#### 问题 1.2.2: 依赖冲突

**问题描述**:
```
go: github.com/ugorji/go/codec@v1.2.7 used for two different module paths (github.com/ugorji/go and github.com/ugorji/go/codec)
```

**解决方案**:

1. 在 go.mod 中添加 replace 指令:
```go
replace github.com/ugorji/go => github.com/ugorji/go v1.2.7
replace github.com/ugorji/go/codec => github.com/ugorji/go/codec v1.2.7
```

2. 整理依赖:
```bash
go mod tidy
```

#### 问题 1.2.3: 依赖版本不兼容

**问题描述**:
```
go: github.com/kataras/blocks@v0.0.8 requires go >= 1.25
```

**解决方案**:

1. 在 go.mod 中添加 replace 指令强制使用兼容版本:
```go
replace github.com/kataras/blocks => github.com/kataras/blocks v0.0.8
replace github.com/kataras/golog => github.com/kataras/golog v0.1.11
```

2. 或者升级 Go 版本到 1.25+

### 1.3 编译问题

#### 问题 1.3.1: CGO 相关错误

**问题描述**:
```
# github.com/mattn/go-sqlite3
./sqlite3-binding.c:25:10: fatal error: 'sqlite3.h' file not found
```

**解决方案**:

1. 安装 SQLite 开发包:


**解决方案**:

```bash

```

```bash
CGO_ENABLED=0 go build -ldflags="-s -w" -o goadmin .
```

## 2. 数据库问题

### 2.1 连接问题

#### 问题 2.1.1: 数据库连接失败

**问题描述**:
```
Error 1045: Access denied for user 'goadmin'@'localhost' (using password: YES)
```

**解决方案**:

1. 检查数据库用户名和密码:
```bash
mysql -u goadmin -p
```

2. 检查数据库配置:
```json
{
  "databases": {
    "default": {
      "host": "localhost",
      "port": "3306",
      "user": "goadmin",
      "pwd": "correct_password",
      "name": "goadmin"
    }
  }
}
```

3. 检查数据库用户权限:
```sql
GRANT ALL PRIVILEGES ON goadmin.* TO 'goadmin'@'localhost';
FLUSH PRIVILEGES;
```

#### 问题 2.1.2: 数据库端口被占用

**问题描述**:
```
Error 2002: Can't connect to local MySQL server through socket '/var/run/mysqld/mysqld.sock' (2)
```

**解决方案**:

1. 检查数据库服务状态:
```bash
sudo systemctl status mysql
```

2. 启动数据库服务:
```bash
sudo systemctl start mysql
```

3. 检查端口占用:
```bash
sudo netstat -tlnp | grep 3306
```

4. 如果端口被占用，终止占用进程:
```bash
sudo kill -9 <PID>
```

### 2.2 表结构问题

#### 问题 2.2.1: 表不存在

**问题描述**:
```
Error 1146: Table 'goadmin.users' doesn't exist
```

**解决方案**:

1. 导入数据库结构:
```bash
mysql -u goadmin -p goadmin < data/admin.sql
```

2. 或者使用 Make 命令:
```bash
make import-mysql
```

3. 验证表是否存在:
```bash
mysql -u goadmin -p goadmin -e "SHOW TABLES;"
```

#### 问题 2.2.2: 字段类型不匹配

**问题描述**:
```
Error 1366: Incorrect string value: '\xF0\x9F...' for column 'username' at row 1
```

**解决方案**:

1. 修改表字符集为 utf8mb4:
```sql
ALTER TABLE users CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 修改字段字符集:
```sql
ALTER TABLE users MODIFY COLUMN username VARCHAR(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.3 性能问题

#### 问题 2.3.1: 查询慢

**问题描述**:
页面加载缓慢，数据库查询时间长。

**解决方案**:

1. 开启慢查询日志:
```sql
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 2;
```

2. 分析慢查询:
```bash
sudo tail -f /var/log/mysql/slow-query.log
```

3. 添加索引:
```sql
CREATE INDEX idx_username ON users(username);
CREATE INDEX idx_email ON users(email);
```

4. 优化查询:
```sql
-- 使用 EXPLAIN 分析查询
EXPLAIN SELECT * FROM users WHERE username = 'admin';

-- 避免 SELECT *
SELECT id, username, email FROM users WHERE username = 'admin';

-- 使用 LIMIT 限制结果数量
SELECT * FROM users LIMIT 100;
```

#### 问题 2.3.2: 连接池耗尽

**问题描述**:
```
Error 1040: Too many connections
```

**解决方案**:

1. 增加数据库连接数:
```sql
SET GLOBAL max_connections = 500;
```

2. 配置连接池:
```json
{
  "databases": {
    "default": {
      "max_idle_conns": 10,
      "max_open_conns": 100
    }
  }
}
```

3. 使用连接池:
```go
engine.SetMaxIdleConns(10)
engine.SetMaxOpenConns(100)
engine.SetConnMaxLifetime(time.Hour)
```

## 4. 认证与权限问题

### 4.1 登录问题

#### 问题 4.1.1: 登录失败

**问题描述**:
用户无法登录，提示用户名或密码错误。

**解决方案**:

1. 检查用户是否存在:
```sql
SELECT * FROM users WHERE username = 'admin';
```

2. 重置密码:
```go
import "golang.org/x/crypto/bcrypt"

hashedPassword, _ := bcrypt.GenerateFromPassword([]byte("new_password"), bcrypt.DefaultCost)
// 更新数据库中的密码
```

3. 检查密码加密方式是否一致:
```go
// 确保使用相同的加密方式
err := bcrypt.CompareHashAndPassword([]byte(hashedPassword), []byte(password))
```

#### 问题 4.1.2: 验证码错误

**问题描述**:
验证码验证失败。

**解决方案**:

1. 检查验证码配置:
```go
admin.SetCaptcha(map[string]string{
    "type": "math",
    "length": "6",
})
```

2. 检查验证码存储:
```go
// 确保验证码正确存储和验证
captcha.Verify(captchaID, captchaValue)
```

3. 检查验证码过期时间:
```go
// 验证码通常 5 分钟过期
// 检查系统时间是否正确
```

### 4.2 权限问题

#### 问题 4.2.1: 权限不足

**问题描述**:
用户访问某个页面时提示权限不足。

**解决方案**:

1. 检查用户角色:
```sql
SELECT r.* FROM users u
JOIN user_roles ur ON u.id = ur.user_id
JOIN roles r ON ur.role_id = r.id
WHERE u.username = 'admin';
```

2. 检查角色权限:
```sql
SELECT p.* FROM roles r
JOIN role_permissions rp ON r.id = rp.role_id
JOIN permissions p ON rp.permission_id = p.id
WHERE r.name = 'admin';
```

3. 分配权限:
```sql
INSERT INTO role_permissions (role_id, permission_id)
SELECT r.id, p.id
FROM roles r, permissions p
WHERE r.name = 'admin' AND p.slug = 'user.create';
```

4. 清除权限缓存:
```go
auth.ClearPermissionCache()
```

#### 问题 4.2.2: 菜单不显示

**问题描述**:
用户登录后菜单不显示。

**解决方案**:

1. 检查菜单权限:
```sql
SELECT m.* FROM menus m
JOIN role_menus rm ON m.id = rm.menu_id
JOIN user_roles ur ON rm.role_id = ur.role_id
JOIN users u ON ur.user_id = u.id
WHERE u.username = 'admin';
```

2. 分配菜单权限:
```sql
INSERT INTO role_menus (role_id, menu_id)
SELECT r.id, m.id
FROM roles r, menus m
WHERE r.name = 'admin' AND m.title = '用户管理';
```

3. 清除菜单缓存:
```go
menu.ClearCache()
```

### 5.2 静态资源问题

#### 问题 5.2.1: 静态资源 404

**问题描述**:
访问静态资源时返回 404 错误。

**解决方案**:

```go
// 配置 CDN 地址
config.AssetUrl = "https://cdn.example.com"
```

3. 启用浏览器缓存:
```nginx
location ~* \.(css|js|jpg|jpeg|png|gif|ico)$ {
    expires 30d;
```

## 8. 安全问题

### 8.1 SQL 注入

#### 问题 8.1.1: SQL 注入漏洞

**问题描述**:
存在 SQL 注入漏洞。

**解决方案**:

1. 使用参数化查询:
```go
// 错误的方式
query := fmt.Sprintf("SELECT * FROM users WHERE username = '%s'", username)

// 正确的方式
query := "SELECT * FROM users WHERE username = ?"
rows, err := db.Query(query, username)
```

2. 使用 ORM:
```go
engine.Where("username = ?", username).Get(&user)
```

3. 输入验证:
```go
// 验证用户名
if !isValidUsername(username) {
    return errors.New("invalid username")
}
```

### 8.2 XSS 攻击

#### 问题 8.2.1: XSS 漏洞

**问题描述**:
存在 XSS 跨站脚本攻击漏洞。

**解决方案**:

1. 输出转义:
```go
import "html/template"

func RenderUser(user *User) template.HTML {
    return template.HTML(fmt.Sprintf("<div>%s</div>", template.HTMLEscapeString(user.Username)))
}
```

2. 使用 CSP:
```nginx
add_header Content-Security-Policy "default-src 'self' http: https: data: blob: 'unsafe-inline'" always;
```

3. 输入过滤:
```go
import "github.com/microcosm-cc/bluemonday"

policy := bluemonday.UGCPolicy()
clean := policy.Sanitize(userInput)
```

### 8.3 CSRF 攻击

#### 问题 8.3.1: CSRF 漏洞

**问题描述**:
存在 CSRF 跨站请求伪造漏洞。

**解决方案**:

1. 使用 CSRF Token:
```go
// 生成 Token
token := csrf.GenerateToken()

// 验证 Token
if !csrf.ValidateToken(token) {
    return errors.New("invalid csrf token")
}
```

2. 使用 SameSite Cookie:
```go
http.SetCookie(w, &http.Cookie{
    Name:     "session",
    Value:    sessionID,
    SameSite: http.SameSiteStrictMode,
})
```

3. 验证 Referer:
```go
referer := r.Header.Get("Referer")
if !isValidReferer(referer) {
    return errors.New("invalid referer")
}
```

## 9. 日志与监控问题

### 9.1 日志问题

#### 问题 9.1.1: 日志文件过大

**问题描述**:
日志文件占用过多磁盘空间。

**解决方案**:

1. 配置日志轮转:
```json
{
  "logger": {
    "rotate": true,
    "rotation_size": 100,
    "rotation_time": 24,
    "max_backups": 7,
    "max_age": 30,
    "compress": true
  }
}
```

2. 使用 logrotate:
```bash
# /etc/logrotate.d/goadmin
/opt/goadmin/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0644 goadmin goadmin
}
```

3. 定期清理旧日志:
```bash
# 添加到 crontab
0 0 * * * find /opt/goadmin/logs -name "*.log" -mtime +7 -delete
```

#### 问题 9.1.2: 日志不输出

**问题描述**:
日志不输出到文件。

**解决方案**:

1. 检查日志目录权限:
```bash
ls -la /opt/goadmin/logs
```

2. 修改目录权限:
```bash
chmod 755 /opt/goadmin/logs
chown goadmin:goadmin /opt/goadmin/logs
```

3. 检查日志配置:
```json
{
  "logger": {
    "rotate": true,
    "log_path": "/opt/goadmin/logs"
  }
}
```

监控数据不准确或不更新。

```go
// 确保监控正确配置
```

## 10. 其他问题

### 10.1 端口被占用处理

**问题描述**:
启动服务时端口被占用。

**解决方案**:

1. 检查端口占用:
```bash
sudo netstat -tlnp | grep 8080
```

2. 查看占用进程:
```bash
sudo lsof -i :8080
```

3. 终止占用进程:
```bash
sudo kill -9 <PID>
```

4. 或者更换端口:
```bash
./goadmin -p 8081
```

### 10.2 磁盘空间不足

**问题描述**:
磁盘空间不足导致服务无法正常运行。

**解决方案**:

1. 检查磁盘空间:
```bash
df -h
```

2. 清理日志文件:
```bash
find /opt/goadmin/logs -name "*.log" -mtime +7 -delete
```

3. 清理临时文件:
```bash
rm -rf /tmp/*
```

4. 清理 Docker 镜像:
```bash
docker system prune -a
```

### 10.3 时间同步问题

**问题描述**:
系统时间不准确导致验证码过期等问题。

**解决方案**:

1. 检查系统时间:
```bash
date
```

2. 同步时间:
```bash
# Ubuntu/Debian
sudo apt-get install ntpdate -y
sudo ntpdate pool.ntp.org

# CentOS/RHEL
sudo yum install ntpdate -y
sudo ntpdate pool.ntp.org
```

3. 配置自动同步:
```bash
# 添加到 crontab
0 */1 * * * /usr/sbin/ntpdate pool.ntp.org
```
